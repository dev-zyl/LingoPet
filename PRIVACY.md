# 隐私说明

LingoPet 主要把设置和用户数据保存在本地。只有在用户启用或使用相关功能时，应用才会请求外部服务。

## 本地保存的数据

应用可能会在 `localStorage` 中保存以下内容：

- API endpoint 和模型名
- 对话模式、人格模式和自定义人格
- 用于 GitHub 提交监控的 GitHub 用户名
- TODO、备忘和提醒数据
- 当前宠物选择、窗口置顶、宠物大小、音量等偏好
- 创意工坊分享时填写的作者昵称

在 Tauri 桌面运行时，API Key 会通过 Rust 后端保存到操作系统 keyring。浏览器开发环境或旧版本迁移场景可能使用 `localStorage`。

导入、编辑或生成的宠物资源会保存在本机，包括项目宠物、生成图片和精灵图。

## 外部请求

根据用户启用的功能，应用可能请求以下服务：

- 用户自己配置的 OpenAI 兼容接口：用于 AI 对话、意图识别和 API 连通性测试。
- GitHub public user / public events 接口：用于提交监控。应用只读取公开事件。
- 一言接口 `https://v1.hitokoto.cn`：用于获取语录文本。
- 宠物市场和创意工坊资源接口，例如 `https://codexpet.xyz`、GitHub Raw、jsDelivr，以及 LingoPet 配置的社区 API。
- 当用户主动点击分享创意工坊动作时，请求 `https://api.lingopet.xyz/api/share`。

应用不会把用户的 API Key 发送给 LingoPet 服务器。API Key 只会在测试或使用对话功能时发送给用户自己配置的 endpoint。

## 文件拖入行为

用户把文件拖到宠物身上时，应用会通过 Tauri 后端把这些文件移动到系统回收站/废纸篓。请在拖入前确认文件确实要删除。

## Windows 音频感知

Windows 下，LingoPet 可以读取系统音频峰值，用于判断当前是否有声音播放，从而驱动宠物音乐律动动画。应用不会录音，不会保存音频，也不会上传音频内容。

## 创意工坊分享

用户主动分享创意工坊动作时，应用会发送动作元数据、作者昵称、可选描述/提示词、帧信息和生成图片数据到社区分享 API。分享成功后，这些内容可能被写入公开 GitHub 创意工坊仓库，并通过公开 CDN 分发。
