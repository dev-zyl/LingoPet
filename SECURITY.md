# 安全政策

## 漏洞反馈

如果你发现安全问题，请尽量先通过非公开方式联系维护者，再公开完整细节。

如果暂时没有私密联系方式，可以先在 GitHub Issue 中提交一个最小化描述，并说明希望通过私密渠道继续沟通。请不要在公开 Issue 中粘贴可直接利用的漏洞代码、私钥、Token、用户隐私数据或其它敏感信息。

普通的安全加固建议、权限收敛建议、依赖升级建议，可以直接通过 GitHub Issue 或 Pull Request 提交。

## 安全相关行为说明

LingoPet 是一个 Tauri 桌面应用，部分功能需要网络访问或系统能力：

- 用户可以配置任意 OpenAI 兼容接口，用于对话、意图识别和连通性测试。
- 应用会从远程宠物市场和创意工坊加载宠物资源、动作补丁和预览图。
- CSP 允许 `http:` / `https:` 图片和连接来源，这是为了支持用户自定义 API 端点、远程宠物资源和社区资源。除非功能明确需要，请不要继续放宽 CSP。
- 拖入宠物窗口的文件可以通过 Tauri 后端移动到系统回收站。
- Windows 下会读取系统音频峰值，用于驱动音乐律动动画；应用不会录音、保存音频或上传音频内容。
- Tauri 桌面运行时会把 API Key 存入系统 keyring。浏览器开发环境或旧版本迁移场景可能使用 `localStorage`。

## 发布签名

GitHub Actions 发布流程需要配置 Tauri updater 签名密钥：

- `TAURI_SIGNING_PRIVATE_KEY`
- `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`

没有配置这些仓库密钥的 fork 仍然可以正常本地开发，运行 `npm run build` 和 `cargo check` 也不需要这些密钥。但推送 `v*` tag 触发发布 workflow 时，会在签名密钥校验步骤失败。

## 反馈范围

欢迎反馈以下范围内的安全问题：

- 桌面应用本体
- Tauri 命令与权限配置
- 宠物 zip 导入、解压和本地文件处理
- updater 配置和发布签名流程
- 创意工坊和社区 API 集成

第三方服务自身故障、用户自填 API Key 在外部服务中的滥用、与 LingoPet 无关的外部服务漏洞，通常不属于本项目安全反馈范围，除非 LingoPet 的实现额外放大了用户风险。
